Node vs Element in the DOM

Definition

Node: A generic representation of any item in the DOM tree, including elements, text, comments, attributes, and more.

Element: A specific type of node that represents HTML or XML tags like <div>, <p>, etc., with nodeType === 1.

Important Points

  • Everything in the DOM is a node.
  • Elements are a type of node (element nodes).
  • All elements are nodes, but not all nodes are elements.
  • Node types include: Document, Element, Text, Comment, Attribute, DocumentFragment, etc.
  • Whitespace, comments, and text are nodes but not elements.

Examples (Code)

Working with Nodes:


// Access all child nodes (includes elements, text, comments)
let allNodes = element.childNodes;

// Create different types of nodes
document.createTextNode("Hello");
document.createComment("This is a comment");
document.createElement("div"); // element node

Working with Elements:


let div = document.createElement("div");
div.className = "box";
div.innerHTML = "This is an element";
document.body.appendChild(div);

All Types of Nodes

  • Document (nodeType 9) - Represents the entire HTML or XML document
  • DocumentType (nodeType 10) - e.g., <!DOCTYPE html>
  • Element (nodeType 1) - HTML tags like <div>, <p>
  • Text (nodeType 3) - The text inside elements
  • Comment (nodeType 8) - Comments like <!-- comment -->
  • Attribute (nodeType 2) - Attributes like id="main" (rarely used as node)
  • DocumentFragment (nodeType 11) - A lightweight container for DOM manipulation

Difference Table: Node vs Element

nodeType Varies: 1 (element), 3 (text), 8 (comment), etc. Always 1
Includes Elements, text, comments, document, etc. Only HTML/XML tag elements
Common Properties nodeName, nodeType, nodeValue id, className, innerHTML

Use Case Full DOM traversal or manipulation User interface interactions & styling

Conclusion

  • The DOM is a tree of nodes, each representing part of the document.
  • An element is a node that specifically represents an HTML tag.
  • All elements are nodes, but not all nodes are elements.
  • Examples of non-element nodes include <!DOCTYPE html>, comments, and text nodes.
